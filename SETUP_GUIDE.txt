# Physical AI & Humanoid Robotics Course - Setup Guide

## Project Overview
This is a comprehensive course on Physical AI and Humanoid Robotics featuring an integrated AI chatbot. The project consists of a Docusaurus frontend and FastAPI backend with RAG (Retrieval-Augmented Generation) capabilities.

## Prerequisites

### System Requirements
- Node.js (v20 or higher)
- Python (v3.8 or higher)
- npm (comes with Node.js)
- Git

### Development Tools
- Git Bash (or any terminal for command execution)
- Code editor (VS Code, etc.)

## Frontend (Docusaurus Course Site) Setup

### 1. Navigate to the website directory:
```bash
cd D:\hackathon01\website
```

### 2. Install dependencies:
```bash
npm install
```

### 3. Start the development server:
```bash
npm start
```
The course will be available at `http://localhost:3000`

### 4. For production build:
```bash
npm run build
```

## Backend (FastAPI Chatbot) Setup

### 1. Navigate to the backend directory:
```bash
cd D:\hackathon01\backend
```

### 2. Install Python dependencies:
```bash
pip install -r requirements.txt
```

### 3. Required Environment Variables (create a .env file in the backend directory):
```
QDRANT_URL=your_qdrant_url
QDRANT_API_KEY=your_qdrant_api_key
OPENAI_API_KEY=your_openai_api_key
```

### 4. Run the backend server:
```bash
python main.py
```
The backend will be available at `http://localhost:8000`

### 5. Backend API Endpoints:
- GET `/` - Root endpoint
- POST `/api/chat` - Chat functionality with RAG
- POST `/api/robot-control` - Robot command processing
- GET `/api/health` - Health check

## Complete Project Setup Process

### 1. Frontend Setup
1. Open Git Bash
2. Navigate to the website directory:
   ```bash
   cd D:\hackathon01\website
   ```
3. Install dependencies:
   ```bash
   npm install
   ```
4. Start frontend server:
   ```bash
   npm start
   ```

### 2. Backend Setup
1. Open a new Git Bash window/tab
2. Navigate to backend directory:
   ```bash
   cd D:\hackathon01\backend
   ```
3. Create virtual environment (optional but recommended):
   ```bash
   python -m venv venv
   venv\Scripts\activate  # On Windows
   ```
4. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```
5. Create `.env` file with API keys:
   ```
   QDRANT_URL=your_qdrant_url
   QDRANT_API_KEY=your_qdrant_api_key
   OPENAI_API_KEY=your_openai_api_key
   ```
6. Run the backend:
   ```bash
   python main.py
   ```

## Testing the Complete System

### 1. Verification Steps
1. Ensure both frontend (port 3000) and backend (port 8000) servers are running
2. Visit `http://localhost:3000` in your browser
3. Navigate through the course chapters
4. Test the "Course Assistant" chatbot at the bottom of each page
5. Ask questions about the course content to test the RAG functionality

### 2. Expected Behavior
- Course content should display properly in Docusaurus
- Chatbot should respond to queries about course content
- Selected text context should be considered in responses
- Robot command processing should work for VLA examples

## Troubleshooting

### Common Issues and Solutions

1. **Frontend build errors:**
   - Clear cache: `npm run clear`
   - Reinstall dependencies: `rm -rf node_modules && npm install`

2. **Backend not connecting to APIs:**
   - Verify API keys are correctly set in the .env file
   - Check network connectivity
   - Verify API quotas haven't been exceeded

3. **Chatbot not responding:**
   - Ensure backend server is running
   - Check browser console for API errors
   - Verify API keys are valid

4. **GitHub Pages deployment:**
   - Workflow file is at `.github/workflows/deploy.yml`
   - The site will be deployed to `https://YOUR_USERNAME.github.io/physical-ai-humanoid-robotics-course/`
